Cairn is an AI-driven automation system that turns natural-language task descriptions into fully autonomous, multi-step security assessments. This guide walks you through everything you need to go from zero to running your first task.

## Prerequisites

Before you begin, make sure you have the following in place.

<CardGroup cols={2}>
  <Card title="Python 3.10+" icon="python">
    Cairn requires Python 3.10 or later. Run `python --version` to check your current version.
  </Card>
  <Card title="LLM provider API key" icon="key">
    You need an API key from a supported provider such as OpenAI, Anthropic, or a compatible local model endpoint.
  </Card>
  <Card title="Git" icon="code-branch">
    Git is required to clone the repository. Most systems have it pre-installed.
  </Card>
  <Card title="Pentest tools (optional)" icon="wrench">
    Tools like `nmap`, `ffuf`, `sqlmap`, and `gobuster` extend Cairn's capabilities but are not required to get started.
  </Card>
</CardGroup>

<Info>
  Cairn can operate with only an LLM API key. External tools like `nmap` and `ffuf` unlock additional capabilities but are optional for your first run.
</Info>

## Get up and running

<Steps>
  <Step title="Clone the repository">
    Clone Cairn from GitHub and navigate into the project directory.

    ```bash
    git clone https://github.com/oritera/Cairn.git
    cd Cairn
    ```

    <Note>
      The repository is currently being prepared for open source release. If the clone is unavailable, check the [GitHub page](https://github.com/oritera/Cairn) for the latest status.
    </Note>
  </Step>

  <Step title="Create a virtual environment">
    Isolate Cairn's dependencies from your system Python installation.

    ```bash
    python -m venv .venv
    source .venv/bin/activate   # On Windows: .venv\Scripts\activate
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required Python packages.

    ```bash
    pip install -r requirements.txt
    ```

    This installs Cairn's core dependencies, including the LLM client libraries, tool orchestration framework, and output formatters.
  </Step>

  <Step title="Configure Cairn">
    Copy the example configuration file and add your LLM provider credentials.

    ```bash
    cp config.example.yaml config.yaml
    ```

    Open `config.yaml` in your editor and set at minimum your LLM provider and API key:

    ```yaml
    llm:
      provider: openai         # or: anthropic, openrouter, local
      model: gpt-4o
      api_key: sk-...          # your API key here
    ```

    <Tip>
      For a full breakdown of every configuration option, see the [configuration guide](/configuration).
    </Tip>
  </Step>

  <Step title="Run your first task">
    Launch Cairn with a natural-language task description. The example below performs a network scan and service identification.

    ```bash
    python cairn.py --task "Scan 192.168.1.0/24 for open ports and identify services"
    ```

    Cairn will decompose the task into sub-steps, select appropriate tools, execute them in sequence, and adapt its strategy based on intermediate results.

    <Warning>
      Only run Cairn against networks and systems you own or have explicit permission to test. Unauthorized scanning is illegal.
    </Warning>
  </Step>

  <Step title="View results">
    When the task completes, Cairn writes structured output to the `output/` directory by default.

    ```bash
    ls output/
    # task_2026-04-23_scan_192.168.1.0/
    #   report.md        - Human-readable findings summary
    #   trace.json       - Full reasoning and tool call trace
    #   raw/             - Raw tool output files
    ```

    Open `report.md` for a formatted summary of findings, or inspect `trace.json` to follow Cairn's reasoning step by step.

    <Tip>
      Pass `--verbose` to stream live output to your terminal as Cairn works through the task.
    </Tip>
  </Step>
</Steps>
