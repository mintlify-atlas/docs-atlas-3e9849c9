Cairn is an AI-driven automation system that autonomously plans, executes, and adapts complex security assessments and general problem-solving tasks. Built on a large language model reasoning core with dynamic tool use, Cairn removes the manual, repetitive burden from security testing — enabling professionals to operate at a higher level of abstraction while the system handles execution.

## Competition background

Cairn was developed and validated through high-stakes competitive testing. At the **Tencent Cloud 2nd AI Penetration Challenge (TCH Hackathon)**, the Bytex@起零衍迹实验室 team achieved a perfect score — the only team to **AK (full score)** the competition — and placed **4th overall** in the online rankings.

<Note>
  The source code is currently being prepared for public release. Follow the [GitHub repository](https://github.com/oritera/Cairn) to be notified when it becomes available.
</Note>

## What Cairn is

Cairn is not a scripted scanner or a fixed-workflow tool. It is an autonomous agent that reasons about a goal, decomposes it into steps, selects appropriate tools, executes those steps, and replans when something goes wrong — all within a single continuous loop driven by a large language model.

This architecture makes Cairn applicable to any multi-step problem where reasoning, adaptation, and tool use are required, with security testing as its primary domain.

## Who Cairn is for

Cairn is designed for practitioners who need reliable, repeatable automation across complex multi-step workflows.

<AccordionGroup>
  <Accordion title="Security researchers and red teamers" icon="shield-halved">
    Cairn automates reconnaissance, vulnerability discovery, and exploitation chaining for penetration testing engagements. It can carry out assessments that would otherwise require significant manual effort, while preserving full reasoning traces for audit and reporting.
  </Accordion>
  <Accordion title="CTF players" icon="flag">
    Cairn can autonomously work through Capture the Flag challenges by reasoning about challenge structure, selecting the right tools, and chaining exploitation steps — making it effective for both practice and competition.
  </Accordion>
  <Accordion title="AI and ML practitioners" icon="brain">

  </Accordion>
  <Accordion title="Security educators" icon="graduation-cap">
